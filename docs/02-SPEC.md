# 欢乐跳棋 - SPEC

## 1. 项目形态

- 类型：HTML5 Canvas 单页棋类游戏
- 构建：无构建系统，无 `package.json`
- 启动：直接打开 `web/start.html`
- 主游戏页：`web/game.html`

## 2. 联机架构

### 2.1 技术选型

| 组件 | 技术 | 说明 |
|------|------|------|
| 信令服务器 | Golang + gorilla/websocket | 房间管理 + WebRTC 信令交换 |
| P2P 通信 | WebRTC DataChannel | 浏览器间直接通信，不经过服务器 |
| 主机 | 创建房间的浏览器 | 权威游戏状态持有者 |
| 客户端 | 加入房间的浏览器 | 接收状态同步，发送操作 |

### 2.2 架构设计

```
[Browser A - Host] <==WebRTC==> [Browser B - Client]
       |                              |
       |         WebSocket            |
       +---------> [Go Server] <------+
       |    (仅房间管理/信令)          |
       |                              |
[Browser A - Host] <==WebRTC==> [Browser C - Client]
[Browser A - Host] <==WebRTC==> [Browser D - Client]
```

### 2.3 服务器职责

- 创建房间，生成 6 位随机房间码
- 维护房间状态（等待中/游戏中/已满）
- 转发 WebRTC SDP Offer/Answer
- 转发 ICE Candidate
- **不处理任何游戏逻辑**

### 2.4 主机职责

- 维护权威游戏状态
- 验证所有移动合法性
- 广播游戏状态更新给所有客户端
- 处理客户端加入/离开

### 2.5 客户端职责

- 渲染游戏画面
- 发送本地操作到主机
- 接收并应用主机广播的状态
- 显示连接状态

## 3. 权威数据源

棋盘、角区、目标角、编号转换统一以 `web/board_data.js` 为准。

- 中央区：61 格，编号 1-61
- 角区：6 个，每个 10 格，编号 62-121
- 总格数：121
- 坐标：axial `(q, r)`
- 像素转换：

```js
x = HEX_SIZE * Math.sqrt(3) * (q + r / 2)
y = HEX_SIZE * 1.5 * r
```

## 4. 角区与目标

| 角区 | 编号范围 | 目标角 |
|------|----------|--------|
| 角0 | 62-71 | 角3 |
| 角1 | 72-81 | 角4 |
| 角2 | 82-91 | 角5 |
| 角3 | 92-101 | 角0 |
| 角4 | 102-111 | 角1 |
| 角5 | 112-121 | 角2 |

每枚棋子记录自己的起点角 `seatId`。胜负判断使用"该棋子是否进入 `seatId` 的目标角"，而不是简单按玩家编号判断。

## 5. 玩家与角区控制

玩家和角区不是一一映射。一个玩家可以控制多个角区。

开始页将"玩家数量"和"每名玩家控制角区数"分开设置。有效组合要求：

```text
控制者数量 * 每名控制者角区数 <= 6
```

其中 1 人局会自动补 1 名 AI，因此控制者数量按 2 计算。

典型座位分配：

| 玩家数 | 每人角区数 | 角区分配 |
|--------|------------|----------|
| 1人 | 1 | 玩家1: 0，AI1: 3 |
| 1人 | 2 | 玩家1: 0/1，AI1: 3/4 |
| 1人 | 3 | 玩家1: 0/1/2，AI1: 3/4/5 |
| 2人 | 1 | 玩家1: 0，玩家2: 3 |
| 2人 | 2 | 玩家1: 0/1，玩家2: 3/4 |
| 2人 | 3 | 玩家1: 0/1/2，玩家2: 3/4/5 |
| 3人 | 1 | 玩家1: 0，玩家2: 2，玩家3: 4 |
| 3人 | 2 | 玩家1: 0/1，玩家2: 2/3，玩家3: 4/5 |
| 4人 | 1 | 玩家1: 0，玩家2: 1，玩家3: 3，玩家4: 4 |
| 5人 | 1 | 玩家1-5 分别控制 0/1/2/3/4 |
| 6人 | 1 | 每人控制一个同编号角区 |

## 6. 移动规则

- 单步：移动到相邻空格。
- 跳跃：相邻方向有任意棋子，且该方向下一格为空，可跳到下一格。
- 连续跳跃：跳跃后如果仍有跳跃目标，可以继续跳跃，也可以主动结束回合。
- 当前实现采用逐跳交互：玩家每次选择下一跳落点；跳跃中可点击"结束跳跃"或再次点击当前棋子结束回合。

## 7. AI

`web/js/ai.js` 提供轻量评估函数：

- `easy`：距离目标 + 较高随机性
- `medium`：距离目标 + 跳跃奖励 + 低随机性
- `hard`：额外考虑进入目标角、避免进入非目标角

当前未实现完整 minimax。后续若要加强 AI，应在保持 `AIEngine.calculateMove(game, player)` 接口稳定的前提下扩展。

## 10. 验证

### 单机模式

无需服务端，直接打开：

- `web/start.html`
- `web/game.html`
基础检查：

- `BOARD_DATA.cells.length === 121`
- 坐标唯一数量为 121
- 每个角区 10 格
- 玩家数量和每人角区数组合满足容量限制时均能进入游戏

### 联机模式

启动 Go 服务器后：

```bash
cd server && go run main.go
```

访问 `http://localhost:8080/start.html`，选择"联机模式"：

1. 主机创建房间，显示房间码
2. 客户端输入房间码加入
3. WebRTC 连接建立后开始游戏
4. 主机断连则游戏结束
